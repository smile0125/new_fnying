官网，母体，整个项目组（Home 以下简称 HM）。

一、母体HM进化的三阶段：Origin，Matrix，The Queen


1. 起源（Origin）阶段

天使资金募集，最小可行开源LA源代码的创建与部分必须功能的HM源代码（非开源）的实现。

结束标志：最小可行LA源代码在Github上的全部开源。

预期时间：半年

2. 母体（Matrix）阶段

再次资金募集，通过全球宣传，寻找并扶持至少3个语言版本，使用开源代码在3个国家创建同源数字资产管理中心LA。

结束标志：各语言版本均稳定运行，各项LA基本业务达成预期功能指标，为本地客户提供满意的服务。

预期时间：一年

3. 皇后（Queen）阶段

最后一次资金募集，创建LA之间的联通，通过HM发行的VIP账号系统使客户的资产可以通过各LA的联通进行无缝对接。完善HM及LA代码，对LA联通发生的问题及时进行反馈和修复。

结束标志：各LA间资金联通稳定运行半年以上，各项基本业务达成预期功能指标，为全球客户提供满意的服务。

预期时间：一年

二、HM盈利模式：

LA认证管理费，LA资产管理费，LA技术支持费，全球VIP账户年费及其他各项服务费用。

三、分布式数字资产管理中心LA

1. 管理标的：

具体某种数字货币；

其他任意数字资产。

2. 标的单位（基准数字资产）：

由LA自主选择一种数字货币作为标的和准备金。默认选择为采用当前最大市值的数字货币的最小单位为一个基准，既一个比特币（BTC）的亿分之一（Satoshi 聪）。

3. 资格取得和终止

资格获取是开放的，任何人都可以通过从Github下载源代码，只要拥有域名，服务器空间和数据库，每个人都可以搭建属于自己的数字资产管理中心。

何时终止服务亦由管理者自行决定。但终止服务后任何对服务用户的清偿处理或需要承担的法律责任等，与HM无任何关系。

4. 主要业务内容：

4.1 招募或承担数字货币代理商BA

招募或承担数字货币代理商BA。

4.2 招募或承担法币代理商CA

视各国法律情况，LA自身或与银行，支付系统，个人合作承担法币兑换基准数字资产的代理。

4.3 数字货币和数字资产的保管

HM，LA自身，或HM推荐的BA。

4.4 客户身份认证及管理

4.5 内部客户转账处理

4.6 跨LA客户转账处理

4.7 货币代理商BA和CA的管理

HM会为LA提供专门的BA，CA管理平台。LA可通过该管理平台设置和管理BA，CA账号账户及身份验证密钥等信息。LA可通过该管理平台查询BA，CA的具体业务处理日志。LA可通过该管理对有问题的业务做出仲裁处理。

5. 盈利模式：

数字资产管理费，客户数字资产担保的法币贷款利息，其他各项服务费用。

四、LA升级进化的各个阶段：

1. Origin 阶段

HM提供的是Unlimited版本的LA源代码，支持万人规模的用户，千人在线。

2. Matrix 阶段

HM将提供一星版本的LA源代码，支持10万人规模的用户，万人在线。

3. Queen 及以后阶段HM将相继提供

二星版本的LA源代码，支持百万规模的用户，10万在线。

三星版本的LA源代码，支持千万规模的用户，百万在线。

支持更多用户的四星及以上LA版本将由HM辅助，三星LA为主自行研发。

五、分布式数字货币代理商BA（distributed bitcoin_agency）

1. 管理标的：

具体某种数字货币;

LA为BA开设的专用账户。

2. 资格取得和终止

BA必须向LA缴纳一定数量的数字货币保证金才能获得BA资格。具体保证金的金额由LA自行制定政策决定。取得资格后BA将获得为LA提供代理充值提现所必须的BA认证信息（交易密钥）。HM会为BA提供专门的工具，调用API接口方便BA自动化处理用户的充值提现请求。

3. 主要业务内容：

3.1 设置BA充值提现汇率

通过HM提供的专用工具可绑定具体某家数字货币交易所的信息进行自动设置。

3.2 处理用户充值请求

通过HM提供的专用工具可自动处理用户的数字货币充值请求。

3.3 处理用户提现请求

通过HM提供的专用工具可自动处理用户的数字货币提现请求。

4. 盈利模式：

BA的充值提现汇率。

5. 特别说明：

因数字货币跨地区的互联网流动属性，一个有能力的BA可以同时代理全球多家LA的充值提现业务。因此，同时代理多家LA的BA有义务接受HM的统一管理，HM将对符合资质的BA颁发相关认证。

六、分布式法币代理商CA（distributed cash_agency）

1. 管理标的：

具体某种法币；

LA为CA开设的专用账户。

2. 资格取得和终止

同BA，CA必须向LA缴纳一定数量的数字货币保证金才能获得CA资格，具体保证金的金额由LA自行制定政策决定，取得资格后CA将获得为LA提供代理充值提现所必须的CA认证信息（交易密钥），调用API接口自动化或半自动地处理用户的充值提现请求。

3. 主要业务内容：

3.1 设置CA充值提现汇率

通过HM提供的专用工具可绑定具体某家数字货币交易所的信息进行自动设置。

3.2 维护CA自动收款接口

有能力的CA可向LA提供自己独立的自动收款接口，方便CA收款后自动化处理用户的充值请求。

3.3 维护CA自动付款接口

有能力的CA可自行创建和维护自动付款接口，方便CA收到用户提现请求后自动化处理用户的法币转账信息。

3.4 处理用户充值请求

通过HM提供的专用工具可自动或半自动地处理用户的法币转数字货币的充值请求。

3.5 处理用户提现请求

通过HM提供的专用工具可自动或半自动地处理用户的数字货币转法币的提现请求。

4. 盈利模式：

CA的充值提现汇率。

七、普通客户US（user）

1. 账号注册

HM会为LA准备用户多因子认证信息，LA自行选择其中若干项作为用户的注册及认证方式。

2. 充值处理

用户根据自己的支付途径选择LA提供的支持该支付途径进行充值的代理商其中之一，按照该代理商提供的当前汇率进行数字货币充值。如在充值过程中产生争议，将相关证据提交LA进行仲裁。

3. 提现处理

用户根据自己的提现途径选择LA提供的支持该提现途径进行提现的代理商其中之一，按照该代理商提供的当前汇率进行提现。如在提现过程中产生争议，将相关证据提交LA进行仲裁。

4. 转账处理

用户将账户中的金额部分或全部划拨至该LA管理或与该LA联通的其他账户上。LA内部转账根据LA各自政策可能收取部分转账手续费用。跨LA转账视转账双方LA及HM的政策可能收取部分转账手续费用。

5. 其他数字资产托管

托管方式和费用收取方式由LA自行决定。LA的Unlimited版本暂不支持该功能。
